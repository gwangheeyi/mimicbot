# 실제 로봇을 다른 컴퓨터에서 제어하기

시연 구성 문서입니다. **가상(Gazebo)은 앱이 도는 내 컴퓨터에서 모두 실행**하고,
**실물 로봇(OMX-AI)만 로봇이 연결된 다른 컴퓨터에서 실행**하는 경우를 설명합니다.

앱은 로봇과 직접 붙지 않습니다. 각 컴퓨터에 떠 있는 **브리지 서버(HTTP :8000)** 에
명령을 보내고, **web_video_server(:8080)** 에서 영상을 받습니다. 그래서 대상이
바뀌어도 앱이 바라보는 **주소(IP)만** 달라질 뿐 나머지는 똑같습니다.

---

## 1. 전체 구조

```
┌───────────────── 내 컴퓨터 (노트북) ─────────────────┐
│  Flutter 앱  ── "실행 대상" 선택 ──┐                  │
│                                    │                  │
│  Gazebo 가상 (127.0.0.1):          │                  │
│    브리지 서버 :8000                │  같은 공유기(LAN) │
│    web_video_server :8080          │      ↕           │
│    Gazebo + 카메라 + 모션/손모방     │                  │
└────────────────────────────────────┼──────────────────┘
                                      │  HTTP 명령 / MJPEG 영상
                                      ▼
┌──────────────── 로봇 컴퓨터 (다른 컴) ────────────────┐
│  브리지 서버 :8000  ── ROS2 토픽 ──▶  실제 OMX-AI 로봇  │
│  web_video_server :8080  ◀── USB 카메라                │
│  실물 로봇 브링업 + 모션 서버 + 손 모방 노드            │
└────────────────────────────────────────────────────────┘
```

- **Gazebo 가상**을 고르면 → 앱이 `127.0.0.1`(내 컴)로 명령·영상을 주고받습니다.
- **OMX-AI 실물**을 고르면 → 앱이 **로봇 컴의 IP**로 명령·영상을 주고받습니다.

두 브리지 서버는 서로 독립입니다. 앱은 그때그때 고른 대상의 컴퓨터에만 명령을 보냅니다.

---

## 2. 역할 나눔

| | 내 컴퓨터 (노트북) | 로봇 컴퓨터 (다른 컴) |
|---|---|---|
| 실행 | Flutter 앱 | 실제 로봇 |
| 서버 | 브리지 :8000, web_video :8080 | 브리지 :8000, web_video :8080 |
| 브링업 | **Gazebo** 시뮬레이션 + 가상 카메라 | **실물 로봇 드라이버** + USB 카메라 |
| 주소 | `127.0.0.1` | LAN IP (예: `192.168.0.20`) |
| 대상 | "Gazebo 가상" | "OMX-AI 실물" |

> 브리지·web_video·모션 서버·손 모방 노드는 **두 컴퓨터에 똑같이** 있습니다.
> 다른 점은 브링업이 Gazebo냐 실물 드라이버냐, 그리고 카메라가 가상이냐 USB냐 뿐입니다.

---

## 3. 사전 준비 — 네트워크

두 컴퓨터가 **같은 공유기(LAN)** 에 있어야 합니다.

1. **로봇 컴에서 IP 확인**
   ```bash
   hostname -I
   # 예: 192.168.0.20
   ```
   맨 앞 사설 IP(보통 192.168.x.x)를 적어 둡니다.

2. **로봇 컴 방화벽에서 포트 열기** (ufw를 쓸 때)
   ```bash
   sudo ufw allow 8000/tcp   # 브리지 서버
   sudo ufw allow 8080/tcp   # web_video_server
   ```

3. **연결 확인** — 내 컴에서 로봇 컴이 보이는지:
   ```bash
   ping 192.168.0.20
   ```

브리지 서버는 이미 `0.0.0.0`으로 열려 있어 외부 접속을 받고, CORS도 열려 있어
웹(Chrome)에서 앱을 띄워도 됩니다.
설정: [config/app_bridge_config.yaml](../ros2_ws/src/open_manipulator_app_bridge/config/app_bridge_config.yaml)

## 4. 앱 설정 (내 컴) — 딱 한 줄

[flutter_app/lib/config/app_config.dart](../flutter_app/lib/config/app_config.dart) 에서
실물 대상 주소를 **로봇 컴의 IP** 로 바꿉니다.

```dart
// Gazebo 가상: 내 컴에서 시뮬레이션을 돌리므로 그대로 둡니다.
static const String gazeboHost = '127.0.0.1';

// OMX-AI 실물: 로봇이 연결된 컴퓨터의 LAN IP 로 바꿉니다. ← 여기!
static const String omxAiHost = '192.168.0.20';
```

이 한 줄이면 앱의 명령·영상·"Micky 깨우기"가 모두 대상에 맞는 컴퓨터로 갑니다.
어느 대상이 어느 주소를 쓰는지는
[flutter_app/lib/services/robot_target.dart](../flutter_app/lib/services/robot_target.dart)
의 `host` 값이 정합니다.

## 5. 로봇 컴 준비

### 5.1 워크스페이스 빌드

```bash
cd ~/mimicbot/ros2_ws
source /opt/ros/jazzy/setup.bash
colcon build
source install/setup.bash
```

### 5.2 로봇 USB 권한 (한 번만)

USB로 연결한 다이나믹셀을 열려면 `dialout` 권한이나 udev 규칙이 필요합니다.
```bash
sudo usermod -aG dialout $USER   # 로그아웃 후 다시 로그인
# 또는 패키지가 제공하는 udev 규칙 설치 스크립트 사용
```
포트 이름은 보통 `/dev/ttyUSB0` 입니다. `ls /dev/ttyUSB*` 로 확인하세요.

### 5.3 "Micky 깨우기" 목록을 **실물용**으로 바꾸기

로봇 컴의 "Micky 깨우기"는 Gazebo가 아니라 **실제 로봇**을 띄워야 합니다.
로봇 컴의
[config/app_bridge_config.yaml](../ros2_ws/src/open_manipulator_app_bridge/config/app_bridge_config.yaml)
에서 `wake:` 목록을 아래처럼 바꿉니다. (앱은 손대지 않습니다 — 각 컴퓨터가
자기 목록을 갖습니다.)

```yaml
wake:
  log_dir: ~/mimicbot/logs
  services:
    # 실물 로봇 브링업 (Gazebo 대신). 포트는 실제 장치에 맞게.
    - name: robot_bringup
      label: 실물 로봇 브링업
      cmd:
        - ros2
        - launch
        - open_manipulator_bringup
        - open_manipulator_x.launch.py
        - port_name:=/dev/ttyUSB0

    # USB 카메라. 앱이 기대하는 /front_camera/image 토픽으로 리맵합니다.
    - name: camera
      label: USB 카메라
      cmd:
        - ros2
        - run
        - usb_cam
        - usb_cam_node_exe
        - --ros-args
        - -r
        - image_raw:=/front_camera/image
        - -p
        - video_device:=/dev/video0

    # 아래 세 개는 가상과 동일합니다.
    - name: motion_server
      label: 모션 서버
      cmd: [ros2, run, open_manipulator_app_control, motion_server]

    - name: hand_mimic
      label: 손 모방 노드
      cmd: [ros2, run, open_manipulator_app_control, hand_mimic_node]

    - name: web_video
      label: 웹 영상 서버
      cmd: [ros2, run, web_video_server, web_video_server]
```

바꾼 뒤에는 반드시 다시 빌드해야 설치본에 반영됩니다(이 워크스페이스는 심링크가
아니라 복사 설치입니다):
```bash
colcon build --packages-select open_manipulator_app_bridge
source install/setup.bash
```

> **카메라 토픽 주의** — 앱은 `/front_camera/image` 를 요청합니다
> ([app_config.dart](../flutter_app/lib/config/app_config.dart) 의 `cameraTopic`).
> 실물 USB 카메라가 다른 토픽으로 나오면 영상이 안 보입니다. 위 예시처럼
> `image_raw:=/front_camera/image` 로 리맵해 토픽 이름을 맞추세요.

### 5.4 브리지 서버 실행

브리지 서버가 먼저 떠 있어야 앱의 "Micky 깨우기"가 나머지를 띄울 수 있습니다.
[scripts/start_mimicbot.sh](../scripts/start_mimicbot.sh) 가 브리지 서버만 띄웁니다.
```bash
cd ~/mimicbot
./scripts/start_mimicbot.sh
```
확인:
```bash
curl http://localhost:8000/health      # {"status":"ok",...}
```

## 6. 시연 실행 순서

1. **로봇 컴**: 로봇 전원 ON → USB 연결 확인 → `./scripts/start_mimicbot.sh` 로 브리지 서버 실행.
2. **내 컴**: [app_config.dart](../flutter_app/lib/config/app_config.dart) 의 `omxAiHost` 를
   로봇 컴 IP로 설정 → (필요시 Gazebo용 브리지도 내 컴에서 실행) → Flutter 앱 실행.
3. **앱 홈에서 대상 선택**:
   - "OMX-AI 실물" 선택 → **"Micky 깨우기"** 누름 → 로봇 컴에서 실물 브링업·카메라·
     모션·손모방·web_video 가 백그라운드로 뜹니다.
   - 아래 로그 상자에 서비스별 시작/실패가 표시됩니다(복사 가능).
4. 이제 **동작 명령 / 실시간 모방 / 자율 작업** 이 모두 로봇 컴으로 나갑니다.
   - "Gazebo 가상"으로 바꾸면 즉시 내 컴 시뮬레이션으로 전환됩니다.

> ⚠️ **안전** — 실물 로봇은 실제로 움직입니다. "Micky 깨우기"·모방 시작 전에 로봇
> 주변에 사람과 물건이 없는지 확인하세요. 앱은 "OMX-AI 실물" 선택 시 안전 안내를 띄웁니다.

## 7. 검증 · 트러블슈팅

| 증상 | 확인 |
|---|---|
| 앱에서 "서버 연결 오류 / Failed to fetch" | 로봇 컴 브리지가 떠 있나(`curl http://<IP>:8000/health`). IP·포트·방화벽 확인. |
| "명령을 발행했지만 받는 노드가 없습니다" | 로봇 컴에서 모션 서버가 떴나(`ros2 node list`), 컨트롤러가 살아 있나. |
| 영상이 안 보임 | 카메라 토픽이 `/front_camera/image` 인가(`ros2 topic list`). web_video 떴나. 브라우저로 `http://<IP>:8080` 접속해 토픽 목록 확인. |
| 로봇이 안 움직임 | 브링업 로그 확인(`~/mimicbot/logs/robot_bringup.log`), USB 포트·권한(`ls /dev/ttyUSB*`), 포트 이름이 `port_name` 인자와 같은지. |
| 손 모방이 안 됨 | 로봇 컴에 웹캠이 연결됐나(hand_mimic_node는 별도 웹캠으로 손을 인식). `~/mimicbot/logs/hand_mimic.log` 확인. |

로봇 컴에서 유용한 확인 명령:
```bash
curl http://localhost:8000/robot/wake/status   # 깨운 서비스들 실행 상태
ros2 node list
ros2 topic list
tail -f ~/mimicbot/logs/*.log
```

## 8. 요약 체크리스트

# 미키(MimicBot) — Claude 지침과 사용법

메뉴 3 "자율 작업"에서 Claude가 어떻게 쓰이는지, 어떻게 실행하고 고치는지 정리한 문서입니다.

---

## 0. 실행 대상 먼저 고르기 (Gazebo 가상 / OMX-AI 실물)

앱을 켜면 **홈 화면 맨 위에서 실행 대상을 먼저 고릅니다.** 여기서 고른 대상으로
세 메뉴(동작 명령·실시간 모방·자율 작업)가 **모두** 동작합니다.

| 대상 | 내용 |
|---|---|
| **Gazebo 가상** | LeRobot을 Gazebo에서 가상으로 실행. 하드웨어 없이 안전하게 시험. **기본값** |
| **OMX-AI 실물** | Robotis OMX-AI 실제 로봇으로 시연. 고르면 안전 안내가 함께 뜹니다. |

기본값이 가상이라 실수로 실물이 움직이지 않습니다.
지금 어느 대상인지는 **각 화면 오른쪽 위 배지**로 항상 확인할 수 있습니다.

> ⚠️ **아직 실제 로봇 통신은 붙어 있지 않습니다.** rosbridge 주소와 양쪽 토픽 이름이
> 정해지지 않아, 지금은 대상 선택과 화면 표시만 실제로 동작하고 명령은 화면상으로만
> 진행합니다. 연결 방법은 아래 5-4를 보세요.

---

## 1. 한눈에 보기

미키가 할 수 있는 일을 **미리 자율행동으로 입력해 두고**, 사용자가 말을 하면
Claude가 그것이 어떤 행동을 시키는 말인지 이해해서 로봇을 움직이는 구조입니다.

```
사용자 말  →  Claude 판단  →  ┌ chat        : 대답만 (로봇 안 움직임)
   (음성/텍스트)              ├ confirm     : "이거 맞나요?" 되물음
                              ├ execute     : 확인됨 → 자율주행 실행
                              └ unsupported : 할 수 있는 게 없다고 답함
```

핵심 규칙 두 가지입니다.

- **확인 없이는 절대 움직이지 않습니다.** 자율주행 명령이 목록의 행동과 비슷하면
  먼저 되물어 확인하고, 사용자가 맞다고 해야 실행합니다.
- **목록에 없는 행동은 지어낼 수 없습니다.** Claude가 고를 수 있는 값이
  실제 자율행동 id로 제한되어 있습니다(아래 4-2 참고).

## 2. 지침 파일

Claude에게 주는 지침 **원문은 아래 파일에 저장되어 있습니다.**

```
flutter_app/assets/claude_instructions.md
```

이 문서에 지침 내용을 복사해 두지 않았습니다. 사본을 두면 둘이 서로 어긋났을 때
어느 쪽이 진짜인지 알 수 없고, 앱은 위 파일 하나만 읽기 때문입니다.
지침을 읽거나 고칠 때는 항상 위 파일을 보세요.

지침에 담긴 내용은 이렇습니다.

| 항목 | 내용 |
|---|---|
| 역할 | 이름은 "미키", 사람을 돕는 모방 로봇, 지금은 자율 작업 모드 |
| 말투 | 한국어, 1~2문장, 마크다운·목록·이모지 금지 (음성으로 읽히므로) |
| 판단 규칙 | chat / confirm / execute / unsupported 네 가지 중 하나를 고르는 기준 |
| 세부 규칙 | 확인 없이 execute 금지, 부정하면 실행 안 함, 없는 행동 지어내기 금지 |
| 응대 방침 | 무례하거나 엉뚱한 말은 화내지 말고 침착하고 짧게 넘긴 뒤 도울 일로 돌아옴 |

### 실제로 전달되는 지침 = 지침 파일 + 자율행동 목록

앱은 위 지침 파일을 읽고, 그 뒤에 **자율행동 목록을 코드에서 자동으로 이어 붙여서**
Claude에게 보냅니다. 그래서 지침 파일에는 행동 목록을 적지 않습니다.
행동을 추가해도 지침을 같이 고칠 필요가 없습니다.

**앱에서 직접 확인하는 법**: 자율 작업 화면 오른쪽 위 문서 아이콘(📄)을 누르면
지금 Claude에게 실제로 가고 있는 지침 전문을 그대로 볼 수 있습니다.

## 3. 사용법

### 3-1. 실행하기 (API 키 필요)

키는 소스에 넣지 않고 실행할 때 주입합니다.

```powershell
cd e:\mimicbot\flutter_app
flutter run -d windows --dart-define=ANTHROPIC_API_KEY=sk-ant-...
```

키 없이 실행하면 앱은 켜지지만, 말을 걸었을 때 화면에 이런 안내가 나옵니다.

> API 키가 설정되지 않았습니다.
> 실행 시 --dart-define=ANTHROPIC_API_KEY=sk-ant-... 를 추가하세요.

### 3-2. 미키에게 말 걸기

세 가지 방법이 있고, 앞의 두 가지는 모두 Claude의 판단을 거칩니다.

| 방법 | 동작 |
|---|---|
| **대화 버튼** (마이크) | 음성 인식(STT) → Claude 판단 → 음성 답변(TTS) |
| **아래 입력창** | 텍스트 입력 → Claude 판단 → 음성 답변 |
| **자율행동 버튼** | 확인 없이 그 행동을 **바로 실행** (Claude를 거치지 않음) |

Windows는 음성 인식(STT)을 지원하지 않아 대화 버튼을 누르면 텍스트 입력창이 대신 뜹니다.
음성 합성(TTS)은 Windows에서도 됩니다.

### 3-3. 대화 예시

**자율주행 명령 (확인 후 실행)**

```
나  : 저거 좀 옮겨줄래?
미키: 혹시 물건 옮기기를 말씀하시는 건가요?     ← confirm (아직 안 움직임)
나  : 응 맞아
미키: 네, 지금 할게요.                          ← execute (다섯 단계 주행 시작)
```

**확인을 거부한 경우**

```
나  : 저거 좀 옮겨줄래?
미키: 혹시 물건 옮기기를 말씀하시는 건가요?
나  : 아니 그거 말고
미키: 그럼 어떤 걸 도와드릴까요?                ← 실행하지 않음
```

**자율주행과 무관한 말**

```
나  : 너 이름이 뭐야?
미키: 저는 미키예요. 무엇을 도와드릴까요?       ← chat (로봇 안 움직임)
```

**할 수 없는 일**

```
나  : 커피 좀 타줘
미키: 커피는 아직 못 타요. 대신 물건은 옮겨드릴 수 있어요.   ← unsupported
```

### 3-4. 실행 중 화면

자율주행이 시작되면 로봇 화면 위에 진행 상황이 뜹니다.
끝난 단계는 체크(✓), 지금 단계는 ▶, 남은 단계는 빈 원으로 표시되고,
오른쪽 정지 버튼을 누르면 즉시 멈춥니다.

## 4. 고치는 법

### 4-1. 지침 고치기 (미키의 성격·말투·판단 기준)

`assets/claude_instructions.md` 를 편집하고 앱을 다시 실행하면 됩니다.
코드는 건드리지 않아도 됩니다.

예를 들어 답을 더 길게 하고 싶으면 "말투" 항목의 `1~2문장으로` 부분을 바꾸면 됩니다.

### 4-2. 자율행동 추가하기

`lib/services/autonomous_skills.dart` 의 `all` 목록에 항목 하나만 더하면 됩니다.

```dart
AutonomousSkill(
  id: 'water_plant',                                  // 목록 안에서 유일해야 함
  name: '화분에 물 주기',
  description: '물뿌리개를 들고 화분에 물을 준다.',      // Claude가 의도를 맞출 때 씀
  examples: ['화분에 물 좀 줘', '식물 시들었어'],        // 사용자가 말할 법한 표현
  steps: ['물뿌리개 집기', '화분 앞으로 이동', '물 주기', '원위치로 복귀'],
),
```

이것만 하면 아래가 **모두 자동으로 따라옵니다.**

- Claude에게 주는 지침의 자율행동 목록
- Claude가 고를 수 있는 `skill_id` 후보
- 화면 아래 자율행동 버튼
- 실행 시 단계별 진행 표시

지침 파일이나 화면 코드를 같이 고칠 필요가 없습니다.

### 4-3. 응답 조절하기

`lib/services/claude_service.dart` 의 `effort` 값이 조절 knob입니다. 지금은 `medium`입니다.

| 증상 | 바꿀 값 |
|---|---|
| 응답이 느림 | `low` |
| 엉뚱한 행동을 고르거나 확인 단계를 건너뜀 | `high` |

## 5. 동작 원리 (개발자용)

| 항목 | 값 |
|---|---|
| 모델 | `claude-opus-4-8` |
| 호출 | Anthropic Messages API 직접 호출 (Dart 공식 SDK 없음) |
| 응답 형식 | 구조화 출력 — `output_config.format` (json_schema) |
| effort | `medium` |
| max_tokens | 1024 (짧은 음성 답변 + 작은 JSON) |
| 대화 기록 | 최근 20개 메시지 유지 |

### Claude가 돌려주는 판단 JSON

```json
{
  "reply": "혹시 물건 옮기기를 말씀하시는 건가요?",
  "action": "confirm",
  "skill_id": "pick_and_place"
}
```

`action`과 `skill_id`는 **enum으로 값이 제한되어 있습니다.**
특히 `skill_id`는 실제 자율행동 id + 빈 문자열만 허용하므로,
Claude가 목록에 없는 행동을 지어내는 일이 구조적으로 불가능합니다.

### 대화 기록이 필요한 이유

"응 맞아" 한마디만 보내면 무엇에 대한 긍정인지 알 수 없습니다.
앞서 미키가 던진 확인 질문이 함께 전달되어야 `confirm` → `execute`로 이어집니다.
그래서 매 요청마다 이전 대화를 함께 보냅니다. (API는 상태를 기억하지 않습니다.)

호출이 실패하면 그 발화는 기록에서 제거해서 다음 판단이 오염되지 않게 합니다.

### 5-4. 실제 로봇 통신 붙이기

화면은 `RobotBackend` 인터페이스만 보고 동작하므로, **화면 코드는 고치지 않아도 됩니다.**
`lib/services/robot_backend.dart` 의 두 클래스에서 `// 연결 지점:` 주석이 달린 자리에
실제 토픽 발행을 채우면 됩니다.

| 클래스 | 담당 |
|---|---|
| `GazeboLeRobotBackend` | Gazebo에 띄운 LeRobot |
| `OmxAiBackend` | Robotis OMX-AI 실물 |

채워야 할 메서드는 여섯 개입니다.

| 메서드 | 언제 불리나 |
|---|---|
| `moveToPoint(x, y)` | 동작 명령 — 화면을 탭해 지점 지정 (0~100 정규화 좌표) |
| `playGesture(gesture)` | 동작 명령 — "안녕", "경례" 등 프리셋/텍스트 명령 |
| `startMimic()` / `stopMimic()` | 실시간 모방 — 시작/정지 |
| `runSkillStep(skill, stepIndex)` | 자율 작업 — 자율행동의 각 단계마다 |
| `stop()` | 자율 작업 — 정지 버튼 |

각 메서드는 화면에 표시할 로그 문구를 돌려줍니다. `line(...)` 헬퍼를 쓰면
앞에 대상 이름이 자동으로 붙습니다(예: `[OMX-AI 실물] 동작 명령 → "경례"`).

필요한 정보는 **rosbridge 주소**(예: `ws://localhost:9090`)와 **양쪽 토픽 이름**입니다.
OMX-AI는 실물이므로 속도 제한과 비상 정지를 `OmxAiBackend` 안에서 함께 처리하는 것을 권합니다.

### 테스트

가짜 HTTP 클라이언트를 쓰므로 실제 API 키와 호출 없이 돌아갑니다.
(키가 있어야 `decide()`가 예외를 던지지 않으므로 아무 값이나 넣어줍니다.)

```powershell
flutter test --dart-define=ANTHROPIC_API_KEY=test-key
```

확인하는 것: 지침+목록 조립, 요청 스키마, chat 판단, confirm→execute 흐름과
대화 기록 전달, 거절(refusal) 처리, 호출 실패 시 기록 정리.
